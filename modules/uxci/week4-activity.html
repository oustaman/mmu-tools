Quick Reminder: Nielsen's 10 Usability Heuristics

These are the principles you'll use to evaluate apps. Click each to see examples:

1. Visibility of System Status

Always tell users what's happening (loading, sent, saved)

2. Match Between System & Real World

Use familiar words & concepts, not jargon

3. User Control and Freedom

Let users undo mistakes easily

4. Consistency and Standards

Keep things predictable across the interface

5. Error Prevention

Stop mistakes before they happen

6. Recognition Over Recall

Show options instead of making users remember

7. Flexibility and Efficiency

Shortcuts for experts, guidance for beginners

8. Aesthetic and Minimalist Design

Remove unnecessary clutter

9. Help Users Recognize Errors

Clear error messages with solutions

10. Help and Documentation

Easy-to-find help when needed

Find 3 UX Problems

Look for things that violate the heuristics. For each problem, write down:

A. What's the problem?

Example: "When I tap 'Send', nothing happens for 5 seconds. No spinner, no feedback."

B. Which heuristic does it violate?

Example: "#1 - Visibility of System Status"

C. How would you fix it?

Example: "Show a loading spinner immediately when tapped"

Repeat this for 3 different problems!
