The Method

1 Review design against each of Nielsen's 10 heuristics
2 Identify violations - where design breaks principles
3 Rate severity of each issue (0-4 scale)
4 Document findings with specific examples
5 Suggest improvements for each issue

Severity Rating Scale

0 = Not a usability problem

Everything works as expected

1 = Cosmetic problem only

Fix if time allows, not critical

2 = Minor usability problem

Low priority fix, causes slight inconvenience

3 = Major usability problem

High priority, significantly impacts usability

4 = Usability catastrophe

MUST fix before release, prevents task completion

Example Evaluation

Banking App - Transaction Screen

Heuristic #3: User Control and Freedom

Issue: No way to cancel transaction once "Submit" is pressed. User must complete transaction or close app entirely (losing all data).

Severity: 3 (Major usability problem)

Evidence: Users report accidentally submitting wrong amounts, no undo option available

Recommendation: Add "Cancel" button visible during processing, Add confirmation dialog before final submit
